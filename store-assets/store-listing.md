# memory — Store Listing

## Short description (132 chars max)

Inject your local memory context into Claude, ChatGPT, and Gemini with one click.

---

## Long description (Chrome: ~16 000 chars | Firefox: no hard limit)

**memory** gives every AI the same context — stored locally on your machine.

Every AI session starts from zero. Claude doesn't know what you told Gemini. ChatGPT doesn't know what you decided last week. You keep re-explaining yourself.

This extension fixes that.

---

### How it works

1. Run the memory CLI: `memory daemon start`
2. Click the **⊕ memory** button that appears on supported AI sites
3. Select a scope (global, or a project-specific one)
4. Your context is injected directly into the chat input — ready to send

---

### Features

- **One-click inject** — prepends your memory context to any chat input
- **Scope selector** — maintain independent contexts per project
- **Live status** — green dot = server running, red = offline
- **Works with** Claude.ai, ChatGPT, Gemini
- **100% local** — no cloud, no accounts, no data leaves your machine
- **Pairs with the memory CLI** — store preferences, decisions, project context across sessions

---

### Privacy

This extension communicates exclusively with `localhost:7711` — your own machine.
No data is sent to any external server. No analytics. No tracking.

---

### Requirements

- The [memory CLI](https://github.com/Sakaax/memory) must be installed and running
- Start the server: `memory daemon start` (or `memory ui`)
